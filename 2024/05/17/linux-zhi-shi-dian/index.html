1、目录

centOS7 安装教程：https://blog.csdn.net/qq_45743985/article/details/121152504

在 CentOS 7 中，根目录 / 下的常见文件夹及其用途如下：

  1. /bin：

    • 包含可执行命令的二进制文件。这些命令可用于系统启动和修复，因此通常在单用户模式下也可用。

  2. /boot：

    • 包含引导加载程序文件和内核镜像文件。当计算机启动时，引导加载程序从此处加载操作系统。

  3. /dev：

    • 包含设备文件。在 Linux 中，一切都是文件，因此 /dev 中的文件用于表示系统硬件设备，如硬盘、键盘、鼠标等。

  4. /etc：

    • 包含系统配置文件。这里存放了许多系统的全局配置文件，例如网络配置、用户账户配置等。

  5. /home：

    • 包含用户主目录。每个用户在此目录下有一个自己的子目录，用于存放个人文件和配置。

  6. /lib和**/lib64**：

    • 包含系统库文件。这些文件是用于支持二进制文件运行的动态链接库。

  7. /media：

    • 用于挂载可移动媒体设备，如USB闪存驱动器、光盘等。

  8. /mnt：

    • 通常用于临时挂载文件系统。它是供用户临时挂载其他文件系统的一个标准目录。

  9. /opt：

    • 用于存放可选的（opt）软件包和附加组件。通常情况下，这里存放的是一些不属于操作系统本身，但是对系统整体有用的软件包。

  10. /proc：

    • 提供系统内核和进程信息的虚拟文件系统。它包含许多系统状态信息，如内存使用情况、CPU信息等。

  11. /root：

    • root用户的主目录。与普通用户的主目录不同，root用户的主目录位于 /root。

  12. /run：

    • 用于存放系统运行时产生的临时文件和进程信息。

  13. /sbin：

    • 包含系统管理员使用的系统命令。这些命令通常需要管理员权限才能执行。

  14. /srv：

    • 用于存放服务相关的数据，例如Web服务器的文件。

  15. /sys：

    • 类似于 /proc，是一个虚拟文件系统，包含了关于系统内核的信息。

  16. /tmp：

    • 用于存放临时文件。这些文件在系统重启时通常会被清理掉。

  17. /usr：

    • 包含用户可读取的文件。其中包含了许多系统应用程序和文件。

  18. /var：

    • 包含变化的文件，例如日志文件、缓存文件、临时文件等。此目录通常用于存放经常变化的文件。

这些是 CentOS 7 中根目录下的常见文件夹，每个文件夹都有其特定的用途，有助于组织系统中的各种文件和数据。

2、基本命令

1、/ 目录和 ~ 目录的区别

/ 目录和 ~ 目录是两个不同的概念，具有不同的含义和用途。

  1. / 目录：

    • / 是 Linux 和类 Unix 系统中的根目录。它是整个文件系统的顶层目录，包含了系统中的所有文件和目录。在 Linux 系统中，所有其他目录和文件都是相对于根目录的。例如，/home 目录包含用户的主目录，/usr 目录包含用户安装的软件。

  2. ~ 目录：

    • ~ 是一个特殊的符号，表示当前用户的主目录（home directory）。每个用户登录到系统后，都有一个主目录，用于存放该用户的个人文件和配置。在终端中，~ 会被自动扩展为当前用户的主目录路径。例如，如果当前用户是 user1，那么 ~ 就会被扩展为 /home/user1。

所以，区别在于：

  • / 是整个文件系统的根目录，而 ~ 是当前用户的主目录。
  • / 是绝对路径，始终指向文件系统的根目录，而 ~ 是相对于当前用户的主目录的路径符号，可以根据不同用户指向不同的目录。

2、 常用命令

在 CentOS 中，有许多常用的命令可以用于管理系统、查看文件和目录、网络配置、安全等方面。以下是一些常见的 CentOS 命令：

  1. 系统管理命令：

    • uname：显示系统信息。
    • uptime：显示系统的负载情况和运行时间。
    • top：实时显示系统资源使用情况。
    • df：显示磁盘空间使用情况。
    • du：显示文件或目录的磁盘使用情况。
    • free：显示系统内存使用情况。
    • ps：显示当前进程状态。
    • kill：结束指定进程。
    • shutdown：关闭或重启系统。

  2. 文件和目录管理命令：

    • ls：列出目录内容。
    • cd：切换目录。
    • pwd：显示当前工作目录。
    • mkdir：创建新目录。
    • rm：删除文件或目录。
    • cp：复制文件或目录。
    • mv：移动文件或目录。
    • touch：创建新文件或更新文件的时间戳。
    • cat：查看文件内容。
    • grep：在文件中搜索指定模式。

  3. 用户和权限管理命令：

    • useradd：添加新用户。
    • passwd：设置用户密码。
    • usermod：修改用户属性。
    • userdel：删除用户。
    • groupadd：添加新用户组。
    • chown：修改文件或目录的所有者。
    • chmod：修改文件或目录的权限。
    • su：切换用户身份。

  4. 网络管理命令：

    • ifconfig：显示和配置网络接口信息（CentOS 7 中推荐使用 ip 命令）。
    • ping：测试网络连接。
    • netstat：显示网络连接、路由表和接口统计信息。
    • ss：显示套接字信息。
    • firewall-cmd：配置防火墙规则。

  5. 软件包管理命令：

    • yum：CentOS 7 中的包管理器，用于安装、更新和删除软件包。
    • rpm：RPM 包管理器，用于管理系统上的软件包。

  6. 日志管理命令：

    • journalctl：查看系统日志。
    • tail：显示文件尾部内容。

  7. 其他常用命令：

    • date：显示或设置系统日期和时间。
    • echo：显示文本或将文本输出到文件。
    • watch：周期性执行命令并显示输出。
    • history：显示最近执行的命令历史记录。
    • scp：在本地主机和远程主机之间安全地复制文件。

这些是在 CentOS 中常见的一些命令，用于系统管理、文件操作、用户管理、网络配置等多个方面。

下面对这个结构的各个部分进行解释：

  1. 命令（command）：是要执行的操作或任务的名称。它可以是系统内置的命令，也可以是用户安装的程序或脚本的名称。例如，ls、cd、mkdir 等都是常见的命令。

  2. 选项（options）：是用于修改命令行为或提供额外功能的标志。通常选项前面使用一个短划线（-）或者两个短划线（–）来表示。例如，-l、-a、--help 等。

  3. 参数（arguments）：是命令操作的对象或输入。参数通常是命令的某些设置、要操作的文件、目录或者其他数据。参数的具体内容根据命令的不同而不同。例如，mkdir 命令需要指定要创建的目录名作为参数。

有些命令并不需要选项或参数，但是基本格式依然是 command [options] [arguments]。有些命令可能有多个选项和参数，你可以根据需要组合使用它们。通常，命令的选项和参数的顺序可以灵活变化，但有些命令可能有特定的语法要求。

理解命令的基本格式对于正确使用和理解命令行工具非常重要。

1、rm -rf /*

警告！rm -rf /* 是一个非常危险的命令，它会尝试删除系统中的所有文件和目录。这个命令的含义是：递归地强制删除根目录下的所有文件和目录。

如果你在一个具有足够权限的系统上运行这个命令，它将会毁掉整个操作系统，导致系统不可用，并可能导致数据丢失。因此，绝对不要在生产环境或者任何重要的系统上运行这个命令。

这个命令是一种非常危险的“黑客”技巧，经常被用来恶意破坏系统。如果你无意中运行了这个命令，请立即关闭系统，并尝试从备份或者恢复介质中恢复系统。

4、find命令和grep命令的区别

find 和 grep 是两个不同的命令，它们在功能和用途上有一些区别。

  1. 功能：

    • find 命令用于在指定目录及其子目录中查找文件，并根据指定的条件进行匹配。它可以根据文件名、文件类型、文件大小、修改时间等条件来搜索文件。
    • grep 命令用于在文件内容中搜索包含指定模式的行。它可以搜索单个文件或多个文件，并且支持使用正则表达式来指定搜索模式。

  2. 用途：

    • find 主要用于查找文件和目录，例如查找特定类型的文件、特定大小的文件、特定修改时间的文件等。
    • grep 主要用于在文件内容中搜索指定模式的行，例如查找包含特定关键词的行、匹配特定模式的行等。

  3. 应用场景：

    • 当你想要查找文件或目录时，使用 find 命令更合适。例如，你想要查找系统中所有大小大于 1MB 的日志文件，就可以使用 find 命令。
    • 当你想要在文件内容中搜索特定内容时，使用 grep 命令更合适。例如，你想要在一个日志文件中查找包含特定错误信息的行，就可以使用 grep 命令。

虽然两者都可以用于搜索文件，但它们的功能和用途不同，需要根据具体的需求来选择使用哪个命令。

  • gzip：gzip 命令用于解压以 .gz 后缀形式压缩的文件。例如，要解压名为 file.txt.gz 的文件，可以使用 gzip -d file.txt.gz 或者 gunzip file.txt.gz 命令。

  • bzip2：bzip2 命令用于解压以 .bz2 后缀形式压缩的文件。例如，要解压名为 file.txt.bz2 的文件，可以使用 bzip2 -d file.txt.bz2 或者 bunzip2 file.txt.bz2 命令。

  • unzip：unzip 命令用于解压 ZIP 格式的压缩文件。例如，要解压名为 archive.zip 的 ZIP 压缩文件，可以使用 unzip archive.zip 命令。

这些是常用的压缩和解压命令，在实际使用中，你可以根据具体的需求选择合适的命令。

16、网络管理

网络管理涉及许多方面，包括配置网络接口、管理网络连接、设置路由、防火墙配置等。下面是一些常见的网络管理任务和相关命令：

  1. 配置网络接口：

    • ifconfig：显示和配置网络接口的信息。例如，使用 ifconfig eth0 192.168.1.100 将网络接口 eth0 设置为 IP 地址 192.168.1.100。

  2. 管理网络连接：

    • ping：用于测试与另一台主机的网络连通性。例如，使用 ping 192.168.1.1 测试是否能够与 IP 地址为 192.168.1.1 的主机通信。
    • netstat：显示网络连接、路由表和网络接口信息。例如，使用 netstat -tuln 查看当前打开的 TCP 和 UDP 端口。
    • ss：显示当前套接字（socket）统计信息。类似于 netstat 命令，但更强大和高效。

  3. 设置路由：

    • route：用于查看和设置路由表。例如，使用 route -n 查看当前的路由表。

  4. 防火墙配置：

    • iptables：用于配置 Linux 内核的防火墙规则。例如，使用 iptables -A INPUT -p tcp --dport 22 -j ACCEPT 允许 SSH 连接。

  5. 网络服务管理：

    • systemctl：用于管理系统服务。例如，使用 systemctl start/stop/restart/network.service 来启动/停止/重启网络服务。

这些是在 Linux 系统中进行网络管理时常用的一些命令和工具。要深入了解网络管理，可能需要学习更多关于网络配置、路由、防火墙和网络服务管理的知识。

1、sudo权限

在 Unix 和类 Unix 系统中，sudo 是一个命令，用于允许普通用户以超级用户（或其他授权用户）的身份执行特定的命令。它允许系统管理员授予普通用户执行一些需要特权的任务的权限，而无需将完整的超级用户权限分配给他们。

sudo 的使用通常需要配置相应的策略文件，以定义哪些用户可以以哪些身份执行哪些命令。这些策略文件通常存储在 /etc/sudoers 文件中。这个文件通常只能由超级用户编辑，以确保安全性。

2、文件基本权限

在 Linux 和类 Unix 系统中，文件的基本权限包括读取（Read）、写入（Write）和执行（Execute）权限，它们分别对应文件的所有者、所属用户组和其他用户。

每个文件或目录都有三组权限，分别用字符表示，分别是：

  • r：读取权限。如果对文件拥有读取权限，则可以查看文件的内容；如果对目录拥有读取权限，则可以查看目录的内容列表。
  • w：写入权限。如果对文件拥有写入权限，则可以修改文件的内容；如果对目录拥有写入权限，则可以在目录中创建、删除或重命名文件。
  • x：执行权限。如果对文件拥有执行权限，则可以执行该文件；如果对目录拥有执行权限，则可以进入该目录。

这些权限对应的表示方式如下：

  • -：表示没有相应的权限。
  • r：表示具有读取权限。
  • w：表示具有写入权限。
  • x：表示具有执行权限。
  • s：表示 SetUID 或 SetGID 位已设置。
  • t：表示粘滞位（Sticky Bit）已设置。

例如，一个文件的权限字符串可能是 -rwxr-xr--，表示：

  • 所有者拥有读、写、执行权限。
  • 所属用户组拥有读、执行权限。
  • 其他用户只有读权限。

要查看文件或目录的权限，可以使用 ls -l 命令，它会列出文件的详细信息，包括权限信息。

1、软件的安装

在 Linux 系统中，有几种常见的方法可以安装应用程序和软件：

  1. 包管理器：
    大多数 Linux 发行版都提供了包管理器，用于从官方软件仓库中安装、更新和删除软件包。常见的包管理器包括：

    • APT（Advanced Package Tool）：Debian 和基于 Debian 的发行版（如 Ubuntu）使用的包管理器。常见命令包括 apt-get 和 apt。
    • YUM（Yellowdog Updater, Modified）：Red Hat 和基于 Red Hat 的发行版（如 CentOS、Fedora）使用的包管理器。常见命令包括 yum 和 dnf。
  • Pacman：Arch Linux 使用的包管理器。常见命令包括 pacman。

    • 下载源代码压缩包并解压缩。
    • 进入解压缩后的目录，通常包含一个 configure 脚本。

rpm和dpt——首选还得是yum

在 Linux 中，RPM（RPM Package Manager）和 DPT（Debian Package Tools）是两种不同的软件包管理系统，用于管理和安装软件包。

  1. RPM（RPM Package Manager）：
    RPM 是 Red Hat Linux 及其衍生发行版（如 CentOS、Fedora）中使用的软件包管理系统。它使用 .rpm 格式的软件包文件，提供了一套命令行工具来安装、升级、删除和查询软件包。常见的 RPM 相关命令包括：

    • rpm -i package.rpm：安装一个 RPM 软件包。
    • rpm -U package.rpm：升级一个 RPM 软件包。
    • rpm -e package：删除一个 RPM 软件包。
    • rpm -q package：查询系统中是否安装了指定的 RPM 软件包。
    • rpm -qa：列出系统中安装的所有 RPM 软件包。

  2. DPT（Debian Package Tools）：
    DPT 是 Debian Linux 及其衍生发行版（如 Ubuntu）中使用的软件包管理系统。它使用 .deb 格式的软件包文件，提供了一套命令行工具来安装、升级、删除和查询软件包。常见的 DPT 相关命令包括：

    • dpkg -i package.deb：安装一个 DEB 软件包。
    • dpkg -r package：删除一个 DEB 软件包。
    • dpkg -l：列出系统中安装的所有 DEB 软件包。
    • apt-get install package：通过 APT 包管理器安装一个软件包（通常会自动下载并安装依赖项）。
    • apt-get remove package：通过 APT 包管理器删除一个软件包。

总的来说，RPM 和 DPT 是两种不同的软件包管理系统，用于管理不同发行版的 Linux 系统中的软件包。根据你使用的发行版和软件包的格式，选择适合的命令和工具来管理软件包。
