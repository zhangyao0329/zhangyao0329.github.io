1. 1 Junit单元测试演示

package Junit;

public class StringUtil {
    public static void printNumber(String name) {
        System.out.println("字符串的长度是" + name.length());
    }

    public static int getMaxIndex(String data) {
        if (data == null) {
            return -1;
        }
        return data.length();
    }
}
package Junit;

import org.junit.Assert;
import org.junit.Test;

public class StringUtilTest {
    @Test
    public void testPrintNumber() {
        StringUtil.printNumber("admin");
    }

    @Test
    public void testGetMaxIndex() {
        int max = StringUtil.getMaxIndex("admin");
        System.out.println(max);
// 断言机制，预测值和实际值进行比较。
        Assert.assertEquals("方法内部有bug", 4, max);
    }
}

1.2 常用注解

2、反射

  • 基本作用：可以得到一个类的全部成分进行操作。
  • 可以破坏封装性。
  • 最重要的用途是:适合做java的框架，基本上，主流的框架都会基于反射设计出一些通用的功能。

2.1 第一步:：Class对象

package com.dapixiu;

public class testStudent {
    /**
     * 程序的入口点
     * 该方法演示了如何获取类的信息并通过类名反射创建类的Class对象
     * @param args 命令行参数
     * @throws Exception 如果发生任何异常，例如类加载失败
     */
    public static void main(String[] args) throws Exception {
        // 获取Student类的Class对象
        Class<Student> c1 = Student.class;
        // 输出类的全限定名
        System.out.println(c1.getName());
        // 输出类的简单名
        System.out.println(c1.getSimpleName());
    
        // 通过类的全限定名获取Class对象
        Class<?> c2 = Class.forName("com.dapixiu.Student");
        // 输出类的简单名
        System.out.println(c2.getSimpleName());
    
        // 比较两个Class对象是否相等，即是否表示同一个类
        System.out.println(c1.equals(c2));
    }
}

2.2 第二步：操作类的构造器

package com.dapixiu;

import org.junit.Test;

import java.lang.reflect.Constructor;
import java.util.Arrays;


public class StudentTest {
    @Test
    public void testGetConstructors() throws Exception {
        // 获取Student类的Class对象
        Class<Student> c = Student.class;

        // 获取Student类的所有公共构造函数
        Constructor<?>[] constructors = c.getConstructors();
        // 打印所有公共构造函数的信息
        System.out.println(Arrays.toString(constructors));

        // 获取Student类的所有声明的构造函数，包括公共的、受保护的、默认的和私有的
        Constructor<?>[] declaredConstructors = c.getDeclaredConstructors();
        // 遍历并打印每个构造函数的名称和参数数量
        for (Constructor<?> dc : declaredConstructors) {
            System.out.println(dc.getName() + "--------->" + dc.getParameterCount());
        }

        // 获取Student类的指定构造函数，这里指定为String和Integer参数的构造函数
        Constructor<Student> d = c.getDeclaredConstructor(String.class, Integer.class);
        // 打印该构造函数的信息
        System.out.println(d);
    }
}

3、注解Annotation

  • 就是Java代码里的特殊标记，比如:@Override、@Test等，作用是:让其他程序根据注解信息来决定怎么执行该程序
  • 注意:注解可以用在类上、构造器上、方法上、成员变量上、参数上、等位置处。

3.1 自定义注解

  • 就是自己定义注解。

  • 特殊属性名：value
  • 如果注解中只有一个value属性，使用注解时，value名称可以不写!!

3.2 注解原理

  • 注解的本质是一个接口，所有的注解都继承了Annotation接口。
  • @注解(……）：其实就是一个实现类对象，实现了该注解以及Annotation接口。

3.3 元注解

3.4 注解的应用场景

package com.dapixiu;

import java.lang.annotation.ElementType;
import java.lang.annotation.Retention;
import java.lang.annotation.RetentionPolicy;
import java.lang.annotation.Target;

@Target(ElementType.METHOD) // 注解只能在方法中使用
@Retention(RetentionPolicy.RUNTIME) // 注解可以一直存活
public @interface MYTest {

}
package com.dapixiu;

import java.lang.reflect.InvocationTargetException;
import java.lang.reflect.Method;

public class Main {
    @MYTest
    public void test() {
        System.out.println("======test1===========");
    }

    @MYTest
    public void test2() {
        System.out.println("======test2===========");
    }


    public void test3() {
        System.out.println("======test3===========");
    }

    public static void main(String[] args) throws InvocationTargetException, IllegalAccessException {
        //1.得到类对象
        Class c = Main.class;
        //2.得到类中的全部成员方法
        Method[] methods = c.getDeclaredMethods();
        //3.遍历方法，看是否存在@MYTest注解
        for (Method method : methods) {

            if (method.isAnnotationPresent(MYTest.class)){
//                存在。触发方法执行
                method.invoke(new Main());
            }
        }
    }
}

4、动态代理

4.1 介绍

  • 对象如果嫌身上干的事太多的话，可以通过代理来转移部分职责。

  • 对象有什么方法想被代理，代理就一定要有对应的方法

public static Object newProxyInstance(ClassLoader loader,
                                      Class<?>[] interfaces,
                                      InvocationHandler h) 
参数1:用于指定一个类加载器
参数2:指定生成的代理长什么样子，也就是有哪些方法
参数3:用来指定生成的代理对象要干什么事情

interface Star：

package Proxy;

public interface Star {
    public void dance();

    public String sing(String name);
}
package Proxy;

public class BigStar implements Star{
    private String name;

    public BigStar(String name) {
        this.name = name;
    }

    public String sing(String name) {
        System.out.println(this.name + "正在唱" + name);
        return "谢谢~";
    }

    public void dance() {
        System.out.println(this.name + "正在跳舞");
    }
}
package Proxy;

import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

public class ProxyUtils {
    public static Star createStar(BigStar bigStar) {
        /*
        * Star starProxy = ProxyUtil.createProxy(s);
        * starProxy.sing("及你太美");
        * starProxy.dance;
        * */
        Star StarProxy = (Star) Proxy.newProxyInstance(
                ProxyUtils.class.getClassLoader(),
                new Class[]{Star.class},
                new InvocationHandler() {
                    @Override
                    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
                        // 代理要做的事情，在这里写代码。
                        if (method.getName().equals("sing")) {
                            System.out.println("准备话筒，收钱20W");
                            return method.invoke(bigStar, args);
                        }else if (method.getName().equals("dance")){
                            System.out.println("准备场地跳舞，收钱50w");
                            return method.invoke(bigStar, args);
                        }else {
                            return method.invoke(bigStar, args);
                        }
                    }
                });
        return StarProxy;

    }
}
package Proxy;

public class Test {
    /**
     * 程序的入口点
     * 本方法演示了如何使用代理类来扩展或修改一个类的行为
     * 在这个例子中，我们创建了一个BigStar对象，然后使用ProxyUtils为其创建一个代理对象
     * 通过这个代理对象，我们可以间接调用BigStar对象的方法，并且可以在调用前后添加额外的操作
     *
     * @param args 命令行参数，本例中未使用
     */
    public static void main(String[] args) {
        // 创建一个BigStar对象，名为"杨超越"
        BigStar s = new BigStar("杨超越");
        // 使用ProxyUtils创建BigStar对象的代理对象
        // 这里的代理对象可以看作是BigStar对象的一个包装，通过它可以间接操作BigStar对象
        Star starProxy = ProxyUtils.createStar(s);
        // 通过代理对象调用sing方法，并打印返回的结果
        // 这里展示了如何通过代理间接执行BigStar的sing方法
        String rs = starProxy.sing("及你太美");
        System.out.println(rs);

        // 通过代理对象调用dance方法
        // 这里展示了如何通过代理间接执行BigStar的dance方法
        starProxy.dance();
    }
}
